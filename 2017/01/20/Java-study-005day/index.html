(Java) 자바의 정석 3판 005일차 - 반복문(for, while, do-while), continue와 break

2017-01-20
Java, 반복문, 자바의 정석

반복문

for문

향상된 for문

배열이나 컬렉션을 순회할 때만 사용할 수 있다.
es6의 for of와 유사하다.

while문

for문과 while문은 항상 변환이 가능하다.
for는 반복할 횟수에 포커스를, while은 조건에 포커스를 뒀다.

연습문제 오답

  • char 범위 구하기
    자꾸만 아스키 코드의 함정에 갇혀있다.
    오답은 아닌데 어차피 int로 변환되므로 굳이 아스키코드로 대조하지 않아도
    숫자인지 영어인지 비교가 가능했다.
    ch > 47 && ch < 58가 아니라
    '0' <= ch && ch <='9'
    (ch > 64 && ch < 91) || ( ch > 96 && ch < 123)가 아니라
    ('a' <= ch && ch <= 'z') || ('A' <= ch && ch <= 'Z')

  • boolean형 변수 powerOn가 false일 때 true인 조건식
    한 가지 경우밖에 떠올리지 못했다.

    • !powerOn
    • powerOn == false
    • powerOn != true
  • 정수, 음수 교차 합산 문제
    또 한번 나의 머리를 강타한 문제다.
    내가 굉장히 편협한 사고 방식을 가지고 있는 게 아닐까 하는 생각이 들었다.
    정답을 살짝 보고 나와 다른 것 같아서 아무리 생각해봤지만…
    저자의 발끝만치도 따라오지 못했다.
    정말 반성해야겠다.
     
    1+(-2)+3+(-4)+… 과 같은 식으로 계속 더해나갔을 때, 몇까지 더해야 총합이 100이상이 되는지 구하시오.
     
    내가 생각한 답안

저자가 제시한 답안
s라는 부호 변수를 놓고, 토글시키면서 쓸 줄은 전혀 몰랐다…

  • 두 개의 주사위를 던졌을 때, 눈의 합이 6이 되는 모든 경우의 수
    필요없는 sum을 선언했었다.
    다른 곳에서 쓰지 않는 변수는 따로 캐싱을 하지 않아도 된다.

  • 정수형 변수에 랜덤한 값 담기.

  • 문자를 정수로 바꾸기.
    parseInt가 만능이 아니었다.

  • 정수의 각 자릿수 구하기.
    문자열이면 정수로 바꾸고 진행하는 게 맘 편하다.

continue와 break

이중 반복문이 있을 때 반복문에 이름을 붙이고, continue와 break 뒤에 반복문의 이름을 붙이면 해당 반복문을 탈출한다.
